# 智学 - AI 技术学习助手

**基于 LangGraph + 通义千问 + MySQL + Chroma 构建的生产级智能 Agent**
集成多用户物理隔离长期记忆、增量 RAG 知识库、双重数据持久化、流式对话，支持 Docker 一键容器化部署，已上线云服务器稳定运行。

## ✨ 项目核心亮点

- **✅ 物理隔离多用户记忆**
  每位用户独立 Chroma 向量库目录，物理隔离数据，彻底杜绝跨用户串记忆问题，安全性极高。

- **✅ 增量智能 RAG 知识库**
  基于 MD5 文件哈希校验机制，自动识别新增/修改/删除文档，**仅增量更新**，极大节省嵌入 API 成本。

- **✅ 双引擎数据持久化（Chroma + MySQL）**
  向量库负责智能检索，MySQL 负责结构化备份、日志留存、数据溯源，双重保障数据不丢失。

- **✅ 工业级异常容错机制**
  内置接口重试、超时兜底、空响应处理、密钥/配额/网络错误智能提示，服务稳定不崩溃。

- **✅ 生产级 Docker 部署**
  完整容器编排、统一北京时间、日志轮转、非 Root 安全运行、数据卷持久化，开箱即用。

- **✅ 沉浸式流式对话体验**
  Token 级打字机流式输出，工具调用状态可视化，多标签会话隔离，交互体验对标商业 AI 产品。

## 📌 详细功能介绍

### 🧠 多用户长期记忆系统（物理隔离）
- 为每个用户分配**独立向量存储目录**，从文件层面实现记忆隔离
- 对话全程静默自动提取用户个人信息、偏好、对话关键内容
- 每次问答自动召回相关历史记忆，实现**个性化、连续性 AI 对话**
- 支持自定义记忆过期天数，自动清理过期无效记忆
- 所有记忆同步写入 MySQL，支持永久备份与数据恢复

### 📚 智能 RAG 文档知识库
- 原生支持 **PDF / DOCX / DOC** 主流技术文档解析
- MD5 全局文件哈希检测，实现**零冗余增量更新**
- 智能文本分片 + 重叠补齐策略，保障上下文完整性
- 技术问答低温度推理，回答严谨专业，自动附带来源文档
- 内置缓存机制，避免重复向量化，大幅节约 API 开销

### 💬 流式智能对话系统
- 原生 Token 级流式输出，实时打字机效果
- 工具调用过程可视化：检索知识库、读取记忆均有友好提示
- 浏览器多标签页独立会话隔离，互不冲突
- 区分**日常对话温度**和**RAG 专业问答温度**，场景适配更精准

### 🗄️ MySQL 双重持久化体系
- 记忆数据、全量对话日志**双库同步存储**
- 对话日志自动过期清理（默认保留90天）
- 记忆采用软删除机制，可定期清理废弃数据，兼顾安全与存储
- 完整结构化数据表设计，支持后续后台管理、对话审计、数据统计

### 🛡️ 高可用容错架构
- 网络超时、API 抖动自动重试，提升服务稳定性
- 密钥错误、配额不足、模型异常**中文精准报错提示**
- 全局异常捕获兜底，不会出现程序崩溃、白屏、卡死
- 兼容大模型空响应、非法格式响应，鲁棒性极强

### 🐳 标准化容器化部署
- Docker Compose 一键拉起 **MySQL8.0 + AI 应用**全套服务
- 容器全局时区统一北京时间，彻底解决时间错乱问题
- 优化数据库连接池、超时参数、最大连接数
- 日志大小&数量限制，防止磁盘爆满
- 非 Root 用户运行，符合生产安全规范
- 全部数据挂载宿主机，容器重启**数据永久不丢失**

### 🎨 Streamlit 可视化前端
- 轻量化高颜值 Web 界面，零前端代码依赖
- 支持用户专属 ID 登录、一键切换用户
- 新手友好功能说明，开箱即用
- 适配 PC 端全屏对话体验

## 💻 技术栈

- **Agent 框架**：LangGraph（智能工作流、工具调度、状态管理）
- **大模型 / 嵌入模型**：阿里云通义千问 DashScope
- **向量数据库**：Chroma（记忆+知识库检索）
- **关系型数据库**：MySQL 8.0（数据备份、日志存储）
- **ORM 框架**：SQLAlchemy
- **Web 前端**：Streamlit
- **文档解析**：PyPDF、docx2txt
- **部署方案**：Docker + Docker Compose

## 🚀 快速部署指南

### 方式一：本地开发运行

#### 1. 克隆项目
```bash
git clone https://github.com/haikang123/AI-Agent-Docker.git
cd AI-Agent-Docker
```

#### 2. 安装依赖
```
pip install -r requirements.txt
```


#### 3. 配置说明
复制项目根目录的 `.env.example` 为 `.env`，填写以下内容：
```
DASHSCOPE_API_KEY="你的API密钥"
USER_ID=user1  # 用户专属记忆ID，首次使用请修改为自己的唯一标识,例如 user_xiaoming
```

#### 4. 项目启动
```
streamlit run app.py
```
### 方式二：Docker 一键部署
``` **Docker 部署重要提示**
数据库地址必须填写 db，绝对不能用 127.0.0.1
不要暴露 MySQL 的 3306 端口到公网，保持 docker-compose.yml 中注释状态
所有数据都会自动挂载到宿主机，容器删除 / 重启数据不会丢失
```

#### 1. 准备项目与目录
```
git clone https://github.com/haikang123/AI-Agent-Docker.git
cd AI-Agent-Docker
mkdir -p docs cache chroma_rag_db chroma_memory_db
```
#### 2. 配置 .env
```
复制项目根目录的 `.env.example` 为 `.env`，填写你的 `DASHSCOPE_API_KEY`，其余配置默认无需修改。
```
#### 3. 一键构建启动
```
docker-compose up -d --build
```
#### 4.访问地址：http://localhost:8501

#### 3. 长期记忆使用

无需手动操作，AI 自动提取、自动存储、自动召回

### 常见问题排查

#### 1. 容器构建太慢
```
已内置阿里云源加速，若依旧缓慢，可切换本地运行模式。
```
####  2. MySQL 连接失败
```
Docker 环境数据库地址必须填写 db，不可使用 127.0.0.1
核对账号、密码、库名与 docker-compose.yml 保持一致
```
#### 3. 文档检索无结果
```
确认 docs 目录内文件完好、格式为 PDF/DOCX/DOC
查看应用日志，确认打印「文档同步完成」
```
#### 4. 重启后记忆丢失
```
检查容器数据卷挂载配置是否正常
核对 .env 中记忆目录与挂载路径一致
```
####  5. API 相关报错
```
密钥错误：检查 Key 内容，删除首尾多余空格
配额不足：前往阿里云 DashScope 控制台充值
请求超时：服务器外网波动，程序已配置自动重试
```
####  6. 第一次启动很慢
```
正常现象，第一次启动会下载 Docker 镜像和依赖包，后续启动会在 10 秒内完成。
```
